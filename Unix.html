Mac GDB (Unable to find Mach task port for process-id 1262: (os/kern) failure (0x5).) 问题

  1. 使用/Applications/Utilities/Keychain Access.app创建证书

    钥匙串访问->证书助理->创建证书
    给证书随笔取一个名字，身份类型“自签名根证书”，证书类型“代码签名”，勾选“让我覆盖...”。
    一路确定，到“指定用于该证书的位置”，选择“系统”。创建，然后就成功了。

  2. 设置改证书为“始终信任”

    选择“系统”、“我的证书”，找到刚刚的证书，选择“始终信任”。
    重启系统。gdb就可以正常调试其它应用程序了。

  3. 使用改证书对gdb签名

    codesign -s [证书的名字] ［gdb的完整路径名］

ssh 登录CentOS慢的解决方案

出现问题的时候用ssh -v登录时发现在以下步骤等了很长时间：

debug1: SSH2_MSG_SERVICE_ACCEPT received

Google得到是CentOS查询DNS失败所致，禁用CentOS sshd的DNS查询功能解决.

sudo vim /etc/ssh/sshd_config
UseDNS no
    
sudo systemctl restart sshd

Debian MultiArch

开启64位Debian的multiarch支持

dpkg --add-architecture i386
dpkg --print-foreign-architectures
aptitude update

在64位Debian系统上运行32位程序

aptitude install libstdc++6:i386 libgcc1:i386 zlib1g:i386 libncurses5:i386

在64位Debian系统上编译32位程序

aptitude install g++-multilib libc6-dev-i386
