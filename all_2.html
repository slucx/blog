ssh 登录CentOS慢的解决方案

出现问题的时候用ssh -v登录时发现在以下步骤等了很长时间：

debug1: SSH2_MSG_SERVICE_ACCEPT received

Google得到是CentOS查询DNS失败所致，禁用CentOS sshd的DNS查询功能解决.

sudo vim /etc/ssh/sshd_config
UseDNS no
    
sudo systemctl restart sshd

Debian MultiArch

开启64位Debian的multiarch支持

dpkg --add-architecture i386
dpkg --print-foreign-architectures
aptitude update

在64位Debian系统上运行32位程序

aptitude install libstdc++6:i386 libgcc1:i386 zlib1g:i386 libncurses5:i386

在64位Debian系统上编译32位程序

aptitude install g++-multilib libc6-dev-i386

set http proxy for git

Command to use:

git config --global http.proxy http://127.0.0.1:8123

If you decide at any time to reset tiis proxy and work without proxy:
Command to use:

git config --global --unset http.proxy

Finally, to check the current set proxy:
Command to use:

git config --global --get http.proxy

Add Public SSH Key to Remote Server

If you want to setup SSH keys to allow logging in without a password, you can do so with a sigle commang.

The first thing you'll need to do is make sure you've run the keygen command to generate the keys:

ssh-keygen -t rsa

Then use this command to push the key to the remote server, modifying it to match your serer name.

cat ~/.ssh/id_rsa.pub | ssh user@hostname 'cat >> .ssh/authorized_keys'

NetworkManager 接管网卡

如果需要让NetWorkManger接管网卡的启动和停止，需要如下步骤

  1. 配置文件 /etc/network/interfaces 里去掉想要NetworkManger接管的接口
  2. 配置/etc/NetworkManager/NetworkManager.conf
[main]  
plugins=ifupdown,keyfile

[ifupdown]
managed=false
  1. service network-manager restart
